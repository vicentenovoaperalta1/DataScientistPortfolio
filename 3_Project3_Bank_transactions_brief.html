About the Project

This project uses unsupervised learning to analyze bank transaction data. It applies K-Means clustering to identify client spending patterns for targeted marketing and Isolation Forest to detect unusual transactions, aiding in fraud detection and risk management.

Exploratory Data Analysis


Dataframe 1

Dataframe 1 is a table of transactions. It contains information about each transaction, in 4.260.904 rows and 10 columns. Some of the important columns are the amount of the transaction, if it has been identified as a fraud, the card number, which will be used to merge with the customer table, and many other relevant columns.

Dataframe 2

Dataframe 2 is a table of customers. It contains information about each customer. It has 1.000 rows and 15 columns. Some of the important columns are gender, job, date of birth and cc_num which will be used to merge with the transaction table.

Part 1: clustering to identify client spending patterns for targeted marketing

Part 1 has its own preprocessing, which includes:

Fraud: The relation between fraud and the different merchants was analyzed, to see if maybe frauds were more related to a set of particular merchants. None relation was found. Therefore, fraud transactions were filtered out


Unique customers: transaction data was examined and transformed to determine how many unique customers were there in the transaction table. There are 1.000 customers, but only 972 have transactions.


Categorical data: there are many categorical columns in the transaction dataframe, but not all of those will be used. "Category" has 14 transaction categories, like "entertainment", "shopping_pos" and "groocery_pos". On the other hand, "merchant" column has 21.977 different categories and "job" has 507 categories, so those columns are not going to be used


DOB: DOB was transformed into age, so that its more meaningful and easy to use


Gender: gender column was transformed into numerical categories with One Hot Encoding



ROC Curve

This model shows a very good ROC Curve and its goodness is confirmed by a ROCAUC = 99.4%.



Confusion Matrix

This heatmap shows the correlations between all the numerical variables in the dataset. Many of the independent variables have high correlations between them, so it's probably better not to include all of them in the ML model. Also, it shows that all the variables are negatively correlated to the target variable.



Classification Report

The model achieves a 99% accuracy, with very good f1-scores of 99% and 81%, and succesfully predicts 89% of customers who will be interested in buying a new car.

Conclusion

This project provides valuable insights for executing the marketing campaign. It reduces the number of clients to be contacted from approximately 131,000 to around 800, while still achieving 89% of the sales. These results assume the model's recommendations are applied directly. Alternatively, the model also outputs probabilities for each customer, allowing flexibility in customizing the number of clients to contact.
